## Как запустить локально

### Требования

- Docker и Docker Compose
- DeepSeek API ключ ([получить на platform.deepseek.com](https://platform.deepseek.com))

### 1. Клонировать репозиторий

```bash
git clone <repo-url>
cd nastavnik_course_gen
```

### 2. Создать файл переменных окружения

```bash
cp .env.example .env
```

Открыть `.env` и заполнить обязательные значения:

```env
DEEPSEEK_API_KEY=sk-your-api-key-here   # обязательно
POSTGRES_PASSWORD=your-secure-password  # сменить для безопасности
```

### 3. Запустить все сервисы

```bash
docker compose up --build
```

При первом запуске Docker соберёт образы и поднимет 4 контейнера:

| Сервис    | URL                        | Описание                 |
|-----------|----------------------------|--------------------------|
| Frontend  | http://localhost:3000      | Next.js веб-интерфейс    |
| Backend   | http://localhost:8000      | FastAPI основной API     |
| ML        | http://localhost:8002      | FastAPI сервис генерации |
| Database  | localhost:5433             | PostgreSQL 16            |

> `docker-compose.override.yml` автоматически применяется при локальном запуске: монтирует исходный код в контейнеры (hot-reload) и пробрасывает порт БД на `5433`.

### 4. Проверить работоспособность

```bash
curl http://localhost:8000/health
curl http://localhost:8002/health
```

### Запуск тестов

```bash
# Все тесты
docker compose exec backend pytest
docker compose exec ml pytest

# Или через skill
/test.backend
/test.ml
```

### Работа с БД (миграции)

```bash
docker compose exec backend alembic upgrade head
```

## Основные функции

### Для пользователя (через веб-интерфейс)

1. **Создание профиля студента** — заполнение формы с целями обучения, текущим уровнем знаний, доступным временем и предпочтениями.
2. **Ручной ввод профиля (JSON)** — импорт готового профиля в формате JSON для тестирования.
3. **Запуск генерации трека** — старт автоматического пайплайна B1–B8, который строит персонализированный учебный трек.
4. **Просмотр результата** — визуализация сгенерированного трека: модули, темы, задачи, расписание.
5. **История треков** — список всех ранее сгенерированных треков по профилю.

### Пайплайн генерации (B1–B8)

Алгоритм запускается автоматически после отправки профиля и проходит 8 последовательных шагов:

| Шаг | Блок                   | Описание                                              |
|-----|------------------------|-------------------------------------------------------|
| B1  | Валидация профиля      | Проверка и обогащение данных студента через LLM       |
| B2  | Компетенции            | Формирование целевых компетенций KSA                 |
| B3  | KSA-матрица            | Построение матрицы знаний, навыков, отношений        |
| B4  | Учебные единицы        | Декомпозиция материала на Learning Units             |
| B5  | Иерархия               | Построение дерева зависимостей между темами          |
| B6  | Формулировки задач     | Генерация проблемных заданий в стиле PBL             |
| B7  | Расписание             | Создание персонального учебного расписания           |
| B8  | Финальная валидация    | Проверка качества и согласованности всего трека      |

Каждый шаг отправляет структурированный промпт в DeepSeek API и возвращает валидированный JSON-результат.

## Архитектура

```
┌─────────────────────────────────────────────────────┐
│                     Frontend                        │
│              Next.js 14 + Tailwind CSS              │
│         (profiles / tracks / manual / qa)           │
└────────────────────┬────────────────────────────────┘
                     │ HTTP REST
┌────────────────────▼────────────────────────────────┐
│                    Backend                          │
│              FastAPI + SQLAlchemy 2.0               │
│    profiles / tracks / logs / manual / health       │
└──────────────┬─────────────────┬───────────────────┘
               │ asyncpg         │ HTTP (httpx)
┌──────────────▼──────┐   ┌──────▼──────────────────┐
│     PostgreSQL 16   │   │      ML Service          │
│   (профили, треки,  │   │  FastAPI + DeepSeek API  │
│    логи генерации)  │   │   пайплайн B1–B8         │
└─────────────────────┘   └─────────────────────────┘
```

**Потоки данных:**
- Frontend → Backend: профили и запросы на генерацию
- Backend → ML: задача генерации с профилем студента
- ML → DeepSeek API: промпты для каждого шага пайплайна
- Backend → PostgreSQL: сохранение профилей, треков, логов

## Модули

### `frontend/src/`

```
app/
  page.tsx           # Главная страница (лендинг)
  profiles/          # Список профилей + создание нового
  tracks/            # Просмотр сгенерированных треков
  manual/            # Ручной ввод профиля (JSON)
  qa/                # QA-инструменты для тестирования
components/          # Переиспользуемые React-компоненты
services/            # API-клиент (fetch к backend)
types/               # TypeScript типы и интерфейсы
```

### `backend/src/`

```
api/
  profiles.py        # CRUD профилей студентов
  tracks.py          # Управление треками и запуск генерации
  logs.py            # Логи выполнения пайплайна
  manual.py          # Ручной режим (прямой вызов ML)
  health.py          # Healthcheck endpoint
core/
  config.py          # Настройки через Pydantic Settings
  database.py        # Async SQLAlchemy engine и сессии
models/              # ORM-модели (Profile, Track, Log)
schemas/             # Pydantic-схемы запросов/ответов
services/            # Бизнес-логика, вызов ML-сервиса
```

### `ml/src/`

```
api/
  pipeline.py        # Endpoint запуска полного пайплайна
  steps.py           # Endpoint запуска отдельных шагов
  manual.py          # Ручной режим генерации
  health.py          # Healthcheck endpoint
pipeline/
  b1_validate.py     # B1: валидация и обогащение профиля
  b2_competencies.py # B2: формирование компетенций
  b3_ksa_matrix.py   # B3: KSA-матрица
  b4_learning_units.py # B4: учебные единицы
  b5_hierarchy.py    # B5: иерархия тем
  b6_problem_formulations.py # B6: задачи PBL
  b7_schedule.py     # B7: расписание
  b8_validation.py   # B8: финальная валидация
prompts/             # Промпты для каждого шага (b1–b8)
services/
  deepseek_client.py # Клиент DeepSeek API (async, retry)
  prompt_reader.py   # Загрузка и рендеринг промптов
schemas/             # Pydantic-схемы шагов пайплайна
```
